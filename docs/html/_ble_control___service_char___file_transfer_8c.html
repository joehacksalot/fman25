Detailed Description

BLE File Transfer characteristic implementation.

Handles reception of multi-frame files over the GATT File Transfer characteristic. Supports three payload types:

  • Settings JSON, applied to UserSettings and forwarded to the system.
  • Custom LED sequence JSON, installed into the LedSequences module.
  • Pairing test JSON, used to confirm and persist a new pairId.

Responsibilities:

  • Parse and validate the configuration frame, detect pairing updates, and initialize the transfer context.
  • Receive and assemble data frames into an internal buffer with bounds checking and progress notifications.
  • Verify completeness and process the final file according to type.
  • Reset the transfer context for subsequent transfers.

uint8_t * data,

bool final )

Receive a chunk of file-transfer data over BLE.

Accepts either a configuration frame (frame 0) or a data frame (frame >= 1). On config, initializes the transfer context and updates pairing if needed. On data, copies bytes into the receive buffer, emits progress notifications, and processes the file when all frames are present.

Parameters
thisPointer to BleControl instance.
dataIncoming ATT write payload buffer.
sizeLength of the payload in bytes.
finalIndicates end-of-transfer hint from client (not strictly required; completeness is verified via frame counts).
Returns
ESP_OK on success; ESP_FAIL on invalid frames or bounds errors.

◆ _BleControl_ProcessTransferedFile()

esp_err_t _BleControl_ProcessTransferedFile ( BleControl * this)
static

Process a fully received file according to its type.

Validates JSON and performs one of:

  • Update custom LED sequence and notify completion.
  • Forward settings JSON to NotificationDispatcher for application.
  • Acknowledge pairing success and notify the system. Emits a NOTIFICATION_EVENTS_BLE_FILE_COMPLETE or _FAILED event at the end.
Parameters
thisPointer to BleControl instance.
Returns
ESP_OK if processing succeeded; ESP_FAIL otherwise.
